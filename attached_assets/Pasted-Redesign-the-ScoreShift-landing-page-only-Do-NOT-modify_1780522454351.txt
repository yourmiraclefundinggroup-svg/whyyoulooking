3. Linear-style animation rules
Use Linear-like restraint.

Linear-style motion principles:
- animate layout transitions subtly
- cards fade/slide in when they enter viewport
- no dramatic zooms
- no excessive scroll jacking
- use crisp easing
- cards should feel responsive and precise

Implement:
- section reveal with IntersectionObserver or Framer Motion
- fade from opacity 0 to 1
- translateY(24px) to 0
- duration 600ms
- stagger children by 80-120ms
- viewport once: true
- hover cards translateY(-4px)
- hover shadow increases slightly
- buttons have tiny scale/brightness response
- tabs/links have animated underline or pill transition

Use easing:
cubic-bezier(0.22, 1, 0.36, 1)

Do not animate every element. Only key elements.

4. Autonomous-style animation rules
Autonomous-style motion should focus on one premium hero object.

Autonomous-style principles:
- one focal object carries motion
- surrounding page stays calm
- product preview feels cinematic
- motion feels like a polished product launch

Implement:
- hero command card as the main moving object
- background moves slower than the card
- floating 3D cards in later sections
- soft depth layers
- object shadow moves subtly with the float
- hover tilt max 3 degrees
- no spinning
- no constant bouncing
- no random animated shapes

3D hover:
On mouse move over large cards:
- rotateX max 2deg
- rotateY max 3deg
- translateY max -4px
- smooth spring return
- preserve accessibility with reduced motion
