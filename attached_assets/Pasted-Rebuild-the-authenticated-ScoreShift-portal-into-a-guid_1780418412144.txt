Rebuild the authenticated ScoreShift portal into a guided Credit Action Intelligence application.

IMPORTANT:
This redesign is for the logged-in portal only.
Do NOT redesign the public landing page.
Do NOT create landing page sections, marketing hero sections, sales sections, or generic feature grids.

The portal is an application interface.

Core positioning:
Array provides the credit data.
ScoreShift turns that data into prioritized actions, guided workflows, execution tools, and progress tracking.

The user should always know:
“What should I do next?”

PORTAL STRUCTURE

Main navigation:

* Home
* My Plan
* Dispute IQ
* Debt Navigator
* Protection Center
* Credit Report
* Progress
* Profile

OVERALL FLOW

Login
→ Goal Selection
→ Timeline Selection
→ Credit Analysis
→ ScoreShift Plan
→ Priority Actions
→ Recommended Tool
→ Tool Workflow
→ Progress Tracking

HOME SCREEN

Title:
Your ScoreShift Plan

Top summary card:

* Selected goal
* Readiness percentage
* Timeline
* Progress bar
* Plain-English summary

Example:
Goal: Mortgage Ready
Readiness: 64%
Timeline: 90 days
Summary: “We found 3 high-impact areas that may be limiting your mortgage approval readiness.”

Primary card:
Next Best Action

Example:
“Reduce revolving utilization below 28%.”

Show:

* Why it matters
* Estimated impact
* Recommended tool
* CTA button

Secondary card:
Credit Snapshot

Show compact score access:

* Experian score
* TransUnion score
* Equifax score
* Last updated date

Buttons:

* Review My Scores
* View Full Credit Report

Scores should be easy to access, but they should not dominate the portal.

MY PLAN SCREEN

Show all prioritized recommendations.

Sections:

* High Priority
* Medium Priority
* Low Priority
* Completed Actions

Each action card must include:

* Priority rank
* Issue type
* Why it matters
* Estimated impact
* Difficulty
* Timeline
* Recommended tool
* CTA

Tool logic:

If issue relates to high utilization, balances, payoff planning, open debt, or debt analysis:
Recommend Debt Navigator.

If issue relates to collections, charge-offs, late payments, bankruptcy reporting, public records, inaccurate reporting, unverifiable information, or bureau reporting differences:
Recommend Dispute IQ.

If issue relates to identity risk, exposed personal information, privacy concerns, PIP scan, or monitoring:
Recommend Protection Center.

If user wants scores, alerts, score simulator, or full report:
Send to Credit Report.

DISPUTE IQ

Dispute IQ is the parent workflow for disputes.

Do NOT treat Smart Letter or ScoreShift Mail as separate main navigation products.

Smart Letter and ScoreShift Mail live inside Dispute IQ.

Dispute IQ handles:

* Collections
* Charge-offs
* Late payments
* Bankruptcy reporting
* Public records
* Inaccurate reporting
* Unverifiable information
* Reporting differences

Dispute IQ workflow:

Issue Selected
→ Analyze Reporting
→ Recommend Dispute Path
→ Generate Smart Letter
→ Preview Letter
→ Send Through ScoreShift Mail
→ Track Delivery
→ Track Bureau Response
→ Recommend Follow-Up

Dispute IQ screen should include:

* Issue summary
* Reporting details
* Bureau differences
* Recommended dispute path
* Smart Letter status
* Mail delivery status
* Response tracking
* Follow-up recommendation

CTA examples:

* Analyze Issue
* Generate Smart Letter
* Preview Letter
* Send Letter
* Track Response

VISUAL DESIGN

Use a premium modern app interface inspired by Linear, Spade, and Cleo, but do not copy their landing page structures.

Use inspiration only for:

* Clean spacing
* Product-first hierarchy
* Smooth interaction
* Premium card design
* Calm guided experience

Visual style:

* Calm fintech aesthetic
* Large whitespace
* Rounded modern cards
* Soft borders
* Subtle shadows
* Clean typography
* Clear hierarchy
* Minimal clutter
* Smooth progressive reveal animations

Avoid:

* Generic AI cards
* Giant credit score gauges
* Loud credit repair visuals
* Cheesy arrows
* Overloaded dashboards
* Too many charts
* Traditional credit monitoring UI
* Random icon grids
* Landing page sections inside the portal

CARD SYSTEM

Use product/action cards, not marketing cards.

Each card should have:

* One purpose
* One main CTA
* Clear recommended tool
* Minimal text
* Strong hierarchy

The “Next Best Action” card should be the most prominent card on the Home screen.

MOBILE DESIGN

Mobile must be designed as its own experience, not a shrunken desktop.

Mobile order:

1. Goal / readiness
2. Next Best Action
3. Recommended tool
4. Credit Snapshot
5. Priority actions
6. Progress journey
7. Solutions

Mobile rules:

* Stack cards vertically
* Use large tappable CTAs
* Avoid wide tables
* Keep text short
* Show one recommendation at a time
* Keep Credit Report access available but secondary

Every screen should answer:

* Why does it matter?

This portal should feel like the operating system for credit improvement, not a credit monitoring dashboard.
